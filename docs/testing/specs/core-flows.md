# 核心链路测试规范

## 覆盖链路
- 看题：`GET /api/problem/{pid}`
- 提交：`POST /api/submission`
- 查看结果：`GET /api/submission/{submissionNo}`

## 数据前置
- 测试执行前后统一执行：`src/test/resources/sql/test-reset.sql`
- 固定账号：
  - `10001`：管理员
  - `10002`：普通用户
  - `10004`：第二普通用户（用于越权场景）
- 固定题目：
  - `p-public`：公开题
  - `p-private`：私有题

## 断言要求
- 成功路径：断言 `HTTP 200` + `$.code == 0` + 关键业务字段。
- 异常路径：断言 `HTTP` 状态 + 业务 `code`。
- 状态变更：必须断言数据库结果（例如提交计数、状态机变化）。

## 异常语义约束
- 未登录：`401 / 40100`
- 无权限：`403 / 40300`
- 资源不存在：`404 / 40400`
- 参数错误：`400 / 40000`
